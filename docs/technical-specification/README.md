## 2. Назначение и цели создания системы

### 2.1. Назначение системы
Система предназначена для автоматизации деятельности физических лиц (коллекционеров значков) по учету, каталогизации и организации личных коллекций. Взаимодействие с системой осуществляется через два интерфейса: веб-сайт (основной) и Telegram-бот (вспомогательный).

### 2.2. Цели создания системы
1. **Электронный учет коллекции.**  
   Реализованы функции создания, просмотра, редактирования и удаления карточек значков с возможностью заполнения всех полей, указанных в п. 4.2.1.4.

2. **Систематизация коллекции.**  
   Группировка значков по категориям и наборам, поиск по названию, фильтрация по категории, набору и состоянию.

3. **Наглядное представление коллекции.**  
   Загрузка и просмотр фотографий для каждого значка и набора. Экспорт коллекции в JPEG.

4. **Управление коллекцией через Telegram-бот.**  
   Бот реализует основные функции веб-сайта, адаптированные для текстового интерфейса. Функции, требующие сложного графического редактирования (кадрирование, поворот фото), остаются доступными только на веб-сайте.

## 3. Характеристики объекта автоматизации

### 3.1. Объект автоматизации
Деятельность коллекционера по ведению каталога своей коллекции значков.  
**Входная информация:** текстовые описания, фотографии значков (JPEG/PNG).  
**Выходная информация:** экранные формы со структурированным списком коллекции, детальные карточки значков, результаты поиска, файл изображения коллекции.

### 3.2. Условия эксплуатации
Деятельность осуществляется пользователем в бытовых условиях или на рабочем месте с использованием ПК или мобильного устройства.  
- **Веб-сайт:** браузеры Chrome, Firefox, Safari, Edge.  
- **Telegram-бот:** официальное приложение Telegram при наличии интернет-соединения.

## 4. Требования к системе

### 4.1. Требования к системе в целом

#### 4.1.1. Требования к структуре и функционированию
1. Система строится по клиент-серверной схеме и включает:
   - Серверную часть (Backend) — единое ядро с бизнес-логикой и API.
   - Клиентскую часть «Веб-сайт» — приложение, взаимодействующее с сервером через API.
   - Клиентскую часть «Telegram-бот» — приложение, взаимодействующее с сервером через тот же API.

2. Режимы функционирования:
   - **Штатный режим** — круглосуточная работа.
   - **Профилактические работы** — не чаще 1 раза в месяц, не более 2 часов, в ночное время.

#### 4.1.2. Требования к численности и квалификации персонала
- **Коллекционер:** навыки работы с веб-браузером и Telegram.
- **Администратор:** навыки администрирования (настройка веб-сервера, работа с PostgreSQL из командной строки), базовые знания Python для диагностики ошибок.

#### 4.1.3. Показатели назначения
- **Нагрузка:** не менее 100 одновременных пользователей.
- **Время отклика (веб-сайт):** 95% запросов к API ≤ 1 сек (при нагрузке до 20 пользователей), загрузка страницы каталога с 50 элементами ≤ 2 сек.
- **Время отклика (Telegram-бот):** ≤ 3 сек для простых команд.
- **Доступность:** коэффициент готовности ≥ 97%, восстановление после сбоя ≤ 6 часов.

#### 4.1.4. Требования к надежности
1. **Резервное копирование:** каждые 6 часов (00:00, 06:00, 12:00, 18:00), хранение 7 дней + ежедневные копии за 30 дней.
2. **Восстановление:** не более 4 часов.
3. **Обработка конкурентного доступа:**
   - Стратегия «последнее изменение побеждает» с проверкой `updated_at`.
   - При конфликте — ошибка с сообщением: «Объект был изменён другим пользователем. Обновите страницу и повторите попытку».
   - Все операции в одной транзакции.
   - Повторные запросы от одного пользователя игнорируются.
   - **ML-операции (удаление фона, детекция, поиск по фото) ставятся в очередь Redis.** Пользователь получает ID задачи и может проверить статус. Конфликты редактирования одного объекта обрабатываются отдельно от очереди.

#### 4.1.5. Требования к защите информации от несанкционированного доступа
1. **Регистрация на веб-сайте:**  
   Поля: Email (уникальный), пароль (≥8 символов: латиница, цифры, заглавные/строчные), подтверждение пароля.  
   После отправки — статус «Не подтвержден», ссылка-подтверждение на email (токен 24 часа).

2. **Подтверждение email:** переход по ссылке → статус «Подтвержден».

3. **Аутентификация на веб-сайте:** по email и паролю, только для подтвержденных.

4. **Восстановление доступа:** форма «Забыли пароль?», токен на 1 час.

5. **Аутентификация в Telegram-боте:**  
   - `/link` — генерация 6-значного кода (15 минут).  
   - Код вводится в профиле на сайте.  
   - `/start` — приветствие и инструкция.  
   - Привязка только для подтвержденного email.

6. **Права доступа:**  
   - **Неавторизованный:** только страницы входа и регистрации.  
   - **Авторизованный:** полный доступ к **своей** коллекции.  
   - **Администратор:** + админ-панель (управление пользователями, токенами, статистика).

7. **Создание администратора:**  
   - Первый администратор создаётся вручную через БД (установка `is_admin = true` для выбранного пользователя).  
   - Последующие — через админ-панель.  
   - Форма регистрации не создаёт администраторов.

8. **Логирование действий администратора:**  
   Таблица `admin_logs` (администратор, действие, дата, IP). Хранение 90 дней.

### 4.2. Требования к функциям, выполняемым системой

#### 4.2.1. Функции, доступные через Веб-сайт (полный функционал)

**1. Управление категориями**  
- Создание: название (до 100 символов), описание (до 500, опционально).  
- Редактирование: изменение названия и описания.  
- Удаление: подтверждение действия.

**2. Ведение каталога полных наборов**  
- Создание: название (до 150), описание (до 1000), общее количество значков (целое число).  
- Редактирование: изменение всех полей.  
- Удаление: подтверждение.  
- Привязка/отвязка значков к набору.

**3. Работа с фотографиями**  
- **Загрузка фото значка:**  
  - Форматы: JPEG, PNG.  
  - Размер файла: ≤ 10 МБ.  
  - Разрешение: не менее 500×500 пикселей.  
  - Количество: до 5 фото на один значок.  
- **Загрузка фото набора:**  
  - Форматы: JPEG, PNG.  
  - Размер файла: ≤ 10 МБ.  
  - Количество: 1 фото.  
- **Просмотр:** открытие фото в модальном окне по клику.  
- **Редактирование главного фото значка:**  
  - **Поворот:** кнопки «Повернуть влево» и «Повернуть вправо» — поворот на 90° в соответствующую сторону.  
  - **Кадрирование:** после поворота (или без него) изображение **автоматически масштабируется с сохранением пропорций** до минимальной стороны 300 пикселей, затем **обрезается по центру до квадрата** 300×300 пикселей. Результат сохраняется как главное фото. Пользователь не выбирает область кадрирования.

**4. Учет личной коллекции**  
- **Обязательные поля:** название (до 200 символов).  
- **Необязательные:** описание (до 5000), год выпуска (целое 4-значное число), материал (до 100), состояние (выпадающий список: «Отличное», «Хорошее», «Среднее», «Плохое»), категория, набор.  
- Функции: просмотр, редактирование, удаление.

**5. Поиск и фильтрация**  
- **Поиск:** по вхождению текста в название значка (регистронезависимый).  
- **Фильтрация:** по категории, набору, состоянию (выпадающие списки). Фильтры могут применяться одновременно.

**6. Экспорт данных**  
- Формирование JPEG-файла коллекции:  
  - Ширина: 1200 пикселей.  
  - Превью главных фото всех значков: по 4 в ряд.  
  - Подписи: название значка.  
  - Если у значка нет главного фото, используется плейсхолдер (серый квадрат с текстом «Нет фото»).

**7. Административная панель** (доступна только администраторам, `/admin/`)  
- **Управление пользователями:** добавление (email, пароль), просмотр списка, блокировка/разблокировка, поиск по email.  
- **Статистика по новым пользователям:** график регистраций с выбором интервала (день/неделя/месяц/произвольный), экспорт CSV.  
- **Статистика по посещениям:** график активности для выбранного пользователя или всех пользователей, выбор интервала, экспорт CSV.  
- **Статистика по БД:** общее количество пользователей, значков, категорий, наборов, фотографий; распределение значков по категориям и состояниям (диаграммы).  
- Требования: все графики интерактивные, таблицы с сортировкой, экспорт CSV, доступ только для администраторов, логирование действий.

#### 4.2.2. Функции, доступные через Telegram-бот (адаптированный интерфейс)

**1. Авторизация и привязка**  
- `/start` — приветствие, инструкция, проверка статуса привязки.  
- `/link` — генерация 6-значного кода (15 минут).

**2. Управление категориями**  
- `/categories` — просмотр списка.  
- Кнопка «Создать» — запрос названия и описания.  
- При выборе категории — кнопки «Редактировать», «Удалить». Удаление возможно только если категория пуста.

**3. Управление наборами**  
- `/sets` — просмотр списка.  
- Кнопка «Новый набор» — запрос названия, описания, общего количества.  
- При выборе набора — просмотр состава, кнопка «Привязать значок» (по ID).

**4. Работа с фотографиями**  
- Загрузка при создании/редактировании значка:  
  - Бот отправляет сообщение: «Отправьте фотографии (до 5). Для завершения — /done».  
  - Пользователь отправляет фото по одному, в любом темпе.  
  - При превышении 5 фото — сообщение: «Вы отправили максимум (5). Для завершения — /done».  
  - Первое фото — главное.  
  - После `/done` — загрузка завершается.  
  - Если пользователь отправил `/done` без фото — сообщение: «Вы не отправили ни одной фотографии. Отправьте хотя бы одно фото или отмените /cancel».  
  - Команда `/cancel` на любом шаге отменяет операцию.  
- `/photo <ID>` — бот присылает все фотографии значка.  
- При редактировании — кнопки «Удалить фото», «Сделать главным».

**5. Управление коллекцией**  
- `/add` — пошаговое создание (название, описание (можно пропустить), год, материал, состояние (кнопки), категория (выпадающий список), набор (выпадающий список), фотографии).  
- `/badge <ID>` — просмотр карточки.  
- `/edit <ID>` — пошаговое редактирование полей.  
- `/delete <ID>` — удаление с подтверждением.

**6. Поиск и фильтрация**  
- `/find <текст>` — поиск по названию, результат — список с ID.  
- `/filter` — инлайн-кнопки для выбора категории, набора или состояния → применение фильтра → список значков.

**7. Экспорт данных**  
- `/export` — формирование JPEG-изображения коллекции (как на сайте) и отправка файлом.

### 4.3. Требования к видам обеспечения

#### 4.3.1. Информационное обеспечение
**СУБД:** PostgreSQL

**Состав данных:**
- `users` (id, email, password_hash, telegram_id, is_admin, created_at)
- `categories` (id, name, description)
- `sets` (id, name, description, total_count, photo_path)
- `badges` (id, user_id, category_id, set_id, name, description, year, material, condition, created_at, updated_at)
- `photos` (id, badge_id, file_path, is_main, uploaded_at, processed_path) — `processed_path` используется для хранения обработанных фото (после SAM/YOLO)
- `admin_logs` (id, admin_id, action, ip_address, created_at)
- `user_visits` (id, user_id, ip_address, user_agent, visited_at)

#### 4.3.2. Программное обеспечение
- **Серверная часть:** Python, PostgreSQL, Redis (очередь ML-операций)
- **Клиентская часть (веб-сайт):** HTML, CSS, JavaScript
- **Клиентская часть (Telegram-бот):** Python, библиотека python-telegram-bot

#### 4.3.3. Техническое обеспечение
- Процессор: 2 ядра
- ОЗУ: 4 ГБ
- Диск: 20 ГБ (SSD)
- ОС: Ubuntu 22.04 LTS
- ПО: Python 3.10, PostgreSQL 14

#### 4.3.4. Лингвистическое обеспечение
- Интерфейс и сообщения об ошибках — на русском языке.
- Кодировка UTF-8.
- Техническая документация — на русском языке.

#### 4.3.5. Организационное обеспечение
- Регистрация на сайте, подтверждение email.
- Привязка Telegram-бота через код.
- В состав документации входит руководство пользователя для веб-сайта и список команд для Telegram-бота.
