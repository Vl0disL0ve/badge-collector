## 2. Назначение и цели создания системы

### 2.1. Назначение системы
Система предназначена для автоматизации деятельности физических лиц (коллекционеров значков) по учету, каталогизации и организации личных коллекций. Взаимодействие с системой осуществляется через два интерфейса: веб-сайт (основной) и Telegram-бот (вспомогательный).

### 2.2. Цели создания системы
1. **Электронный учет коллекции.**  
   Реализованы функции создания, просмотра, редактирования и удаления карточек значков.

2. **Систематизация коллекции.**  
   Группировка значков по категориям и наборам, поиск по названию, фильтрация.

3. **Наглядное представление коллекции.**  
   Загрузка и просмотр фотографий, экспорт коллекции в JPEG.

4. **Интеллектуальная обработка изображений.**  
   Удаление фона (SAM), детекция и подсчет элементов (YOLO), поиск по фото.

5. **Управление коллекцией через Telegram-бот.**  
   Бот реализует основные функции веб-сайта, адаптированные для текстового интерфейса.

6. **Многопользовательский режим с системой токенов.**  
   Каждый пользователь работает только со своей коллекцией (псевдо-однопользовательский режим).  
   Ресурсоемкие операции (ML) требуют токенов.

7. **Персонализация и аналитика.**  
   Личный кабинет (профиль, токены, история, избранное).  
   Админ-панель с графиками активности и управления токенами.

## 3. Характеристики объекта автоматизации

### 3.1. Объект автоматизации
Деятельность коллекционера по ведению каталога своей коллекции значков.  
**Входная информация:** текстовые описания, фотографии значков (JPEG/PNG).  
**Выходная информация:** экранные формы со структурированным списком коллекции, детальные карточки значков, результаты поиска, изображение коллекции, результаты интеллектуальной обработки.

### 3.2. Условия эксплуатации
Деятельность осуществляется в бытовых условиях или на рабочем месте с использованием ПК или мобильного устройства.  
**Веб-сайт:** браузеры Chrome, Firefox, Safari, Edge.  
**Telegram-бот:** официальное приложение Telegram при наличии интернет-соединения.

## 4. Требования к системе

### 4.1. Требования к системе в целом

#### 4.1.1. Требования к структуре и функционированию
1. Система строится по клиент-серверной схеме и включает:
   - Серверную часть (Backend) — единое ядро с бизнес-логикой и API.
   - Клиентскую часть «Веб-сайт» — приложение, взаимодействующее с сервером через API.
   - Клиентскую часть «Telegram-бот» — приложение, взаимодействующее с сервером через тот же API.
2. Режимы функционирования:
   - **Штатный режим** — круглосуточная работа.
   - **Профилактические работы** — не чаще 1 раза в месяц, не более 2 часов, в ночное время.

#### 4.1.2. Требования к численности и квалификации персонала
- **Коллекционер:** навыки работы с веб-браузером и Telegram.
- **Администратор:** навыки администрирования (настройка веб-сервера, PostgreSQL, Python).

#### 4.1.3. Показатели назначения
- **Нагрузка:** не менее 100 одновременных пользователей.
- **Время отклика (веб-сайт):** 95% API-запросов ≤ 1 сек (до 20 пользователей), загрузка каталога с 50 элементами ≤ 2 сек.
- **Время отклика (Telegram-бот):** ≤ 3 сек для простых команд.
- **Доступность:** коэффициент готовности ≥ 97%, восстановление после сбоя ≤ 6 часов.

#### 4.1.4. Требования к надежности
1. **Резервное копирование:** каждые 6 часов (00:00, 06:00, 12:00, 18:00), хранение 7 дней + ежедневные копии за 30 дней.
2. **Восстановление:** не более 4 часов.
3. **Обработка конкурентного доступа:**
   - Стратегия «последнее изменение побеждает» с проверкой `updated_at`.
   - При конфликте — ошибка с сообщением.
   - Все операции в одной транзакции.
   - Повторные запросы от одного пользователя игнорируются.

#### 4.1.5. Требования к защите информации от несанкционированного доступа
1. **Регистрация на веб-сайте:**  
   Поля: Email (уникальный), пароль (≥8 символов, латиница, цифры, заглавные/строчные), подтверждение пароля.  
   После регистрации — статус «Не подтвержден», ссылка-подтверждение на email (токен 24 часа).

2. **Подтверждение email:** переход по ссылке → статус «Подтвержден».

3. **Аутентификация на веб-сайте:** по email и паролю, только для подтвержденных.

4. **Восстановление доступа:** форма «Забыли пароль?», токен на 1 час.

5. **Аутентификация в Telegram-боте:**  
   Команда `/link` → 6-значный код (15 минут). Код вводится в профиле на сайте.  
   Команда `/start` — приветствие и инструкция.  
   Привязка только для подтвержденного email.

6. **Права доступа:**  
   - Неавторизованный: страницы входа и регистрации.  
   - Авторизованный: полный доступ к **своей** коллекции.  
   - Администратор: + админ-панель (управление пользователями, токенами, статистика).

7. **Создание администратора:**  
   Первый администратор создается вручную через БД (флаг `is_admin`).  
   Последующие — через админ-панель.  
   Форма регистрации не создает администраторов.

8. **Логирование действий администратора:**  
   Таблица `admin_logs` (администратор, действие, дата, IP). Хранение 90 дней.

#### 4.1.6. Псевдо-однопользовательский режим
Каждый пользователь **видит и управляет только своей коллекцией**.  
Данные других пользователей недоступны для просмотра, поиска или фильтрации.  
Система не предоставляет функционала «социальной сети» (лента, поиск по чужим коллекциям).

### 4.2. Требования к функциям, выполняемым системой

#### 4.2.1. Функции, доступные через Веб-сайт (полный функционал)

**1. Управление категориями**  
Создание, редактирование, удаление (с подтверждением).  
Поля: название (до 100 символов), описание (до 500).

**2. Ведение каталога полных наборов**  
Создание, редактирование, удаление.  
Поля: название (до 150), описание (до 1000), общее количество значков.  
Привязка/отвязка значков к набору.

**3. Работа с фотографиями**  
- Загрузка фото значка: JPEG/PNG, до 10 МБ, мин. 500×500 пикселей, до 5 шт.  
- Загрузка фото набора: до 1 шт.  
- Просмотр в модальном окне.  
- **Редактирование главного фото значка:**  
  - Кадрирование: автоматическая обрезка по центру до квадрата (1:1), результат ≥ 300×300 пикселей.  
  - Поворот: на 90° влево/вправо перед кадрированием.

**4. Учет личной коллекции**  
- **Обязательное поле:** название (до 200 символов).  
- **Необязательные:** описание (до 5000), год выпуска (4 цифры), материал (до 100), состояние (выпадающий список: «Отличное», «Хорошее», «Среднее», «Плохое»), категория, набор.  
- Функции: просмотр, редактирование, удаление.

**5. Личный кабинет пользователя**  
- Просмотр и редактирование профиля (смена пароля).  
- Отображение баланса токенов.  
- **История действий:** добавление, редактирование, просмотры значков, ML-операции.  
- **Избранное:** добавление/удаление значков, просмотр списка.

**6. Интеллектуальная обработка изображений (требует токены)**  
- **Удаление фона** (SAM): загрузка фото → обработка → сохранение результата.  
- **Детекция и подсчет элементов** (YOLO): загрузка фото → вывод количества объектов на изображении.  
- **Поиск по фото:** загрузка изображения → поиск похожих значков в коллекции пользователя.

**7. Поиск и фильтрация**  
- Поиск по вхождению текста в название значка.  
- Фильтрация по категории, набору, состоянию.  
- Фильтры могут применяться одновременно.  
- Результат ограничен коллекцией текущего пользователя.

**8. Экспорт данных**  
Формирование JPEG-файла: ширина 1200 пикселей, превью главных фото всех значков (по 4 в ряд) с подписями (название).

**9. Административная панель** (доступна только администраторам, `/admin/`)  
- **Управление пользователями:** добавление, блокировка/разблокировка, поиск по email.  
- **Управление токенами:** начисление/списание токенов выбранному пользователю.  
- **Статистика по новым пользователям:** график регистраций (интервал: день/неделя/месяц/произвольный), экспорт CSV.  
- **Статистика по посещениям:** график активности (выбранный пользователь или все), экспорт CSV.  
- **Статистика по БД:** общее количество пользователей, значков, категорий, наборов, фотографий; распределение значков по категориям и состояниям (диаграммы).  
- Все графики интерактивные, таблицы с сортировкой, логирование действий.

#### 4.2.2. Функции, доступные через Telegram-бот (адаптированный интерфейс)

**1. Авторизация и привязка**  
- `/start` — приветствие, инструкция, статус привязки.  
- `/link` — генерация 6-значного кода для привязки (15 минут).

**2. Управление категориями**  
`/categories` — просмотр, создание, редактирование, удаление (кнопки).

**3. Управление наборами**  
`/sets` — просмотр, создание, просмотр состава, привязка значка по ID.

**4. Работа с фотографиями**  
- Загрузка при создании/редактировании значка:  
  Бот отправляет сообщение «Отправьте фотографии (до 5). Для завершения — /done».  
  Принимает фото по одному, с паузами.  
  Первое фото — главное.  
  После `/done` — завершение загрузки.  
- `/photo <ID>` — отправляет все фото значка.  
- При редактировании — кнопки «Удалить фото», «Сделать главным».

**5. Управление коллекцией**  
- `/add` — пошаговое создание (название, описание, год, материал, состояние, категория, набор, фотографии).  
- `/badge <ID>` — просмотр карточки.  
- `/edit <ID>` — пошаговое редактирование.  
- `/delete <ID>` — удаление с подтверждением.

**6. Избранное и история**  
- `/favorites` — просмотр избранных значков.  
- `/history` — просмотр последних действий.

**7. Поиск и фильтрация**  
- `/find <текст>` — поиск по названию.  
- `/filter` — инлайн-кнопки для выбора категории, набора или состояния.

**8. Личный кабинет**  
`/profile` — просмотр профиля (email, баланс токенов).

**9. Экспорт данных**  
`/export` — формирование JPEG-изображения коллекции (как на сайте) и отправка файлом.

**10. Интеллектуальная обработка (требует токены)**  
- `/removebg <ID>` — удаление фона у фото значка.  
- `/detect <ID>` — детекция элементов на фото значка.  
- `/findphoto` — поиск по загруженному фото.

### 4.3. Система токенов

- Каждому новому пользователю начисляется **100 токенов**.
- Стоимость операций (списание):
  - Удаление фона (SAM): **5 токенов**
  - Детекция YOLO: **3 токена**
  - Поиск по фото: **2 токена**
- Администратор может изменять баланс пользователя (начислять/списывать).
- При недостатке токенов система возвращает сообщение: *«Недостаточно токенов»* и не выполняет операцию.

### 4.4. Требования к видам обеспечения

#### 4.4.1. Информационное обеспечение
**СУБД:** PostgreSQL

**Состав данных:**
- `users` (id, email, password_hash, telegram_id, is_admin, token_balance, created_at)  
- `categories` (id, name, description)

- `favorites` (id, user_id, badge_id, created_at)  
- `history` (id, user_id, action_type, badge_id, details, created_at)  
- `token_transactions` (id, user_id, amount, operation_type, created_at)  
- `admin_logs` (id, admin_id, action, ip_address, created_at)  
- `user_visits` (id, user_id, ip_address, user_agent, visited_at)

#### 4.4.2. Программное обеспечение
- **Серверная часть:** Python, FastAPI, PostgreSQL, Redis (очередь для ML-операций)

- **Клиентская часть (Telegram-бот):** Python, python-telegram-bot
- **ML-модели:** SAM (сегментация), YOLO (детекция)

#### 4.4.3. Техническое обеспечение
- Процессор: 2 ядра  
- Оперативная память: 4 ГБ  
- Дисковое пространство: 20 ГБ (SSD)  
- ОС: Ubuntu 22.04 LTS  
- ПО: Python 3.10, PostgreSQL 14

#### 4.4.4. Лингвистическое обеспечение
- Интерфейс и сообщения об ошибках — на русском языке.
- Кодировка UTF-8.
- Техническая документация — на русском языке.

#### 4.4.5. Организационное обеспечение
- Регистрация на сайте, подтверждение email.
- Привязка Telegram-бота через код.
- Ресурсоемкие операции (ML) расходуют токены.
